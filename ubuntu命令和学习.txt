Ubuntu学习笔记
ubuntu的安装：
（1）先在windows上清空一个盘装ubuntu，右键我的电脑点管理-磁盘管理-选择一个有剩余的盘，然后选压缩卷，压缩60G。
    如果之前装了ubuntu，那么将装ubuntu的磁盘直接删除（右键然后删除卷）。不需要先将待安装的磁盘再分boot和swap之类的，
    因为进ubuntu安装界面选择挂载的时候会自动分区。等下ubuntu装在这个盘里，系统装好后，在ubuntu里可以看见windows下的盘和里面的内容。
（2）用刻录光盘的软件把下载的iso系统刻录到U盘里，做启动盘。注意先格式化再刻录。
（3）开机按F2进入bios，把U盘设置为第一启动。注意：选择uefi+U盘品牌的名字如：netac（朗科）作为第一启动，千万不要直接选U盘。
（4）分区的时候选择刚分好的盘点+号:   
    1、先分/boot，主分区，起始位置，ext4格式，大概2G。
    2、然后分swap区，交换分区，起始位置。
    3、最后所有空间的都给/，ext4格式。
             



    ls
{
        ls =list,显示目录中的文件夹。
        ls -a = ls -all，表示显示所有文件，包括隐藏文件。
        ls -l =ls -long，显示文件详细信息。
        ll，相当于ls -l，但还显示隐藏文件。详细内容见centos的ls。
}

    cd
{
        cd . 切换到当前目录
        cd .. 切换到上一级目录
        cd - 最近使用的2个目录来回切换。
}



    *，？，~
{
        *可以代替任意多个字符，也可以表示什么都没有。
        ？有且只能代替一个字符。
        ls ab* 显示开头有ab的文件
        ls a?b 显示两头ab中间任意字符的文件。
        ~ 表示家目录。
        cd ~ 切换到家目录。
}

    cp,mv
{
        cp ab /home/paul/Desktop，这样复制如果存在一样名字的文件名那么会直接覆盖。
        cp -i ，加个-i如果有一样名字的会有提示是否覆盖。
        cp -r ，能复制目录。
        mv同理，同一个目录下移动可以改名。
}

    echo，>，>>
{
        echo “hello python” > >123; 表示把hello python写到文件123中，另起一行写到末尾。因为中间有空格所以要加“号。
        >>是不覆盖的，另起一行在末尾添加。
        >是覆盖的，先删除再添加。
        echo后面是插入的内容，以文本的形式插入，如果后面接命令，那就是把命令以文本的形式插入。
            如：echo ls >> 123，则123的末尾会增加ls，而不是ls显示的目录。
        命令 >> 123，则是把命令显示的内容增加到123中。如ls >> 123，是把ls显示的目录添加到123的末尾。
}

    管道符 |
{
        允许一个命令的输出作为另一个命令的输入。
        cat 123 | grep ab 查看123文件，并查找文件中包含ab的内容。
}

    ps
{
        ps = process status 查看当前正在执行的进程。
        a 显示所有进程，包括其他用户的。
        u 显示进程的详细信息。
        x 显示不由终端启动的进程。
        如：ps au 查看正在执行的由终端执行的进程。
}

    top
{
        top 动态的查看当前系统的cpu和内存使用状况。
        按q推出。
        安装htop软件，显示效果更炫酷。
}

    kill
{

        kill [-9] id : 关闭进程的命令，-9表示强制关闭，id表示进程号，要用ps au查看后确定了再执行。
}

    Shell python
{
        这两个是脚本语言，ubuntu都默认安装了shell和python。脚本语言载体是文本文档。
        shell的脚本是以sh结尾的。Python脚本是以py结尾的。
        Shell语言的执行程序在/bin/sh，因此执行sh结尾的脚本文件时，完整的命令是：/bin/sh ./hello （/bin/sh表示shell的路径）。
        也可以直接输入./hello来执行，但可能出现2个问题：
        1、该shell文件的权限不够，因此修改权限。Chmod 766 hello 
            (4代表读，2代表写，1代表执行)7表示读，写，执行都有，6表示只有读和写，4表示只读。
            其中第1个数字代表文件创建者的权限。第2个数字表示所属组。第3个表示其他人。
        2、可能shell或python的环境变量没有设置，故找不到执行程序。
}
